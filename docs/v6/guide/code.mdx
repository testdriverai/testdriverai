TestDriver allows you to execute custom shell scripts within your test workflows using the [`exec`](/commands/exec) command. This feature, introduced in version `5.1.0`, enables you to integrate custom logic, such as generating one-time passwords (OTPs), hitting APIs, or performing other dynamic operations, directly into your tests.

## Key features

1. **Run Shell Scripts**:

- Execute custom shell or PowerShell commands within your test steps.

2. **Dynamic Outputs**:

- Store the result of your script in a variable for use in subsequent steps.

3. **NPM Support**:

- Install and use NPM packages in your scripts.

## Updated example: One-time password (OTP) validator

This example demonstrates how to install a package and run commands using `exec`.

```yaml install-example.yaml
version: 6.0.0
steps:
  - commands:
      - command: exec
        lang: pwsh
        code: |
          npm install -g http-server
      - command: exec
        lang: pwsh
        code: |
          Start-Process pwsh -ArgumentList "-Command", "http-server -p 8080"
```

## Additional details

- The [`exec`](/commands/exec) command now takes a `lang` argument with supported values `sh` or `pwsh`.
- `sh` code is executed in the shell on the runner.
- `pwsh` code is executed in the PowerShell on the runner.

<Note>
The `result` variable is already available in your script, overwrite it to store the output as shown in the example.

The `output`argument is assigned automatically by setting `result = somestringvalue` in the script you run.

</Note>

## Protips

- Always assign the output of your script to the `result` variable.
- Ensure all required NPM packages are installed locally and in the `prerun` script when using GitHub Actions.
